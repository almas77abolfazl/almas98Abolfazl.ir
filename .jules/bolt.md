## 2026-07-21 - CLS Optimization

**Learning:** When using web fonts, they can cause significant CLS if space is not reserved for the text before the fonts finish loading. Even with `display=swap`, a shift can occur when the fallback font is replaced by the loaded font. Furthermore, Google Fonts, while fast, introduce an additional network roundtrip that can delay font loading. By downloading `.woff2` files and serving them locally, along with using `<link rel="preload">`, we can ensure the fastest possible font rendering. Images also require explicit `width` and `height` attributes to allow the browser to calculate the correct aspect ratio and reserve layout space, even if they are styled with CSS like `w-full h-full object-cover`.

**Action:** When implementing a design with web fonts or dynamic images, proactively consider CLS. Add a `min-height` or bounding box to text containers, especially in prominent areas like headers. Always provide explicit `width` and `height` attributes for images, even when relying on CSS for sizing. Prefer serving fonts locally using `.woff2` format, preloading them, and applying `font-display: swap` to minimize render-blocking and layout shifts.
