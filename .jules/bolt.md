## 2024-05-24 - Improve Initial Load Time (Reduce Unused JS)
 **Learning:** Angular 17+ provides `@defer` which is extremely powerful for lazy loading components when they enter the viewport. Also, eagerly loading entry points like `HomeComponent` still increases the initial bundle. It can be beneficial to lazy load the root route (`loadComponent` instead of `component`) to keep the `main.js` chunk smaller, especially when it relies on external libraries. Finally, bumping the build target to `ESNext` in `tsconfig.json` can significantly reduce the amount of transpilation/polyfills for modern browsers, saving kilobyte overhead.
 **Action:** Proactively suggest using `loadComponent` for all routes and `@defer` for below-the-fold UI elements when tasked with reducing initial bundle sizes in Angular 17+ applications. Always ensure the `target` in `tsconfig` reflects modern capabilities unless legacy browser support is explicitly required.
